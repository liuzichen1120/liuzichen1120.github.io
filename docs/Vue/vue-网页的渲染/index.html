网页的渲染

  • 浏览器在渲染页面时，做了哪些事
  1. 加载页面的html和css（源码）
  2. html转换为DOM，css转换为CSSDOM
  3. 将DOM和CSSDOM构建成一颗渲染树
  4. 对渲染树进行reflow(回流，重排)（计算元素的位置）
  5. 对网页进行绘制（repaint）（重绘）
  • 渲染树（Rander Tree）
  1. 从根元素开始检查哪些元素可见，以及他们的样式
  2. 忽略那些不可见的元素（display:none）
  3. 对所有可见元素的样式进行匹配
  • 重排，回流（reflow）

  1. 计算渲染树中元素的大小和位置

  2. 当页面中的元素的大小或位置发生变化时，便会触发页面的重排(回流)

  3. 注意：每次修改这类样式(width，height，margin，font-size)都会触发一次重排！所以如果分次修改多个样式都会触发重排多次，

    而重排是非常耗费系统资源的操作(昂贵)，重排次数过多后，会导致网页的显示性能变差，在开发时我们应该尽量的减少重排的次数

  4. 在现代的前端框架中，这些东西都已经被框架优化过了！所以使用vue，react这些框架开发时，几乎不需要考虑这些问题（通过虚拟DOM）

    唯独需要注意的是，尽量减少在框架中直接操作DOM

  5. 例子

<!DOCTYPE html>
<html lang="zh">
    <head>
        <meta charset="UTF-8" />
        <meta http-equiv="X-UA-Compatible" content="IE=edge" />
        <meta name="viewport" content="width=device-width, initial-scale=1.0" />
        <title>Document</title>
        <style>
            .box1 {
                width: 200px;
                height: 200px;
                background-color: orange;
                /*  display:none*/
            }

            .box2 {
                background-color: tomato;
            }

            .box3 {
                width: 300px;
                height: 400px;
                font-size: 20px;
            }
        </style>
    </head>
    <body>
        <button id="btn">点我一下</button>
        <hr />
        <div id="box1" class="box1"></div>
        <script>
            btn.onclick = () => {
                // box1.classList.add("box2")
                // 可以通过修改class来间接的影响样式，来减少重排的次数
                // box1.style.width = "300px"
                // box1.style.height = "400px"
                // box1.style.fontSize = "20px"
                // box1.classList.add("box3")
                // 可以通过display来减少重排，这样只会重排两次
                // box1.style.display = "none"
                // box1.style.width = "300px"
                // box1.style.height = "400px"
                // box1.style.fontSize = "20px"
                // div.style.display = "block"
            }
        </script>
    </body>
</html>

  • 重绘
  1. 绘制页面
  2. 当页面发生变化时，就会对页面进行重新的绘制
  3. 如果添加display：none属性，页面没有发生变化。点击按钮，只会触发重绘，不会触发重排。

v-if 和v-show在重绘重排没有差别
