事件修饰符

事件修饰符​

  • .stop 停止事件的传播
  • .capture 在捕获阶段触发事件
  • .prevent 取消默认行为（例如表单提交等等）
  • .self 只有事件由自身触发时才会有效

Vue 3 中的 .self 与 Vue 2 中的 .self 的作用是一样的，用于限制触发事件的目标必须是绑定了事件的元素本身，而不是其子元素。举个例子： 

<template>
    <div  @click.self="handleClick">
        <div>子元素</div>
    </div>
</template>

<script>
export  default  {
    methods:  {
        handleClick()  {
            console.log('点击了  div  元素本身')
        }
    }
}
</script>

在上面的代码中，当点击 .self 绑定了 click 事件的 div 元素本身时，才会触发 handleClick 方法，而点击它的子元素则不会触发该方法。

需要注意的是，.self 只能限制直接绑定它的元素本身触发事件，无法阻止冒泡到它的祖先元素上，因此如果需要完全阻止事件冒泡，需要配合.stop 一起使用。

  • .once 绑定一次性的事件

在Vue3中，.once是一个指令用法，它用于在一个元素上绑定一个单次事件监听器。一旦触发了事件，监听器就会被移除，即使绑定的元素再次触发同样的事件，也不会再次执行该监听器。

例如，如果我们想让一个按钮在被点击后只触发一次，我们可以使用.once指令：

<button  @click.once="handleClick">Click  me</button>

在这个例子中，我们把@click.once绑定到按钮上，并把它与一个名为handleClick的方法相关联。一旦按钮被点击，handleClick方法就会被调用，但它仅会被调用一次。即使用户再次点击按钮，handleClick方法也不会再次被调用。

需要注意的是，.once指令只能与事件相关的指令一起使用，例如@click、@keydown等。它不能与其他指令一起使用，例如v-model、v-show等。

  • .passive 主要用于提升滚动事件的性能

Vue3 中的按键修饰符是在组件事件处理程序中使用的一种指令，用于指定事件触发时需要同时按下的键盘按键，以达到特定的功能或行为。

Vue3 提供了多种按键修饰符，包括：

  • .enter - 当按下回车键时触发。
  • .tab - 当按下 Tab 键时触发。
  • .delete/.backspace - 当按下删除键或后退键时触发。
  • .esc - 当按下 Esc 键时触发。
  • .space - 当按下空格键时触发。
  • .up - 当按下上方向键时触发。
  • .down - 当按下下方向键时触发。
  • .left - 当按下左方向键时触发。
  • .right - 当按下右方向键时触发。

使用方式如下：

<template>
    <div>
        <input  type="text"  @keyup.enter="submitForm">
        <button  @click.ctrl="deleteItem">Delete</button>
    </div>
</template>

<script>
export  default  {
    methods:  {
        submitForm()  {
            //  处理表单提交逻辑
        },
        deleteItem()  {
            //  处理删除逻辑
        },
    },
};
</script>

在上述示例中，@keyup.enter 和 @click.ctrl 就是按键修饰符，分别表示在按下回车键（keyup.enter）和按下 Ctrl 键同时点击按钮（click.ctrl）时触发对应的事件处理函数。

需要注意的是，按键修饰符只能用于常规按键，不能用于功能键（如 Shift、Ctrl、Alt 等）。而对于功能键，可以使用 Vue3 的其他指令或者使用 @keydown 事件配合代码处理。


Vue3中的按键修饰符与Vue2中的使用方法相同，都是通过在事件监听器中添加修饰符来实现的。

在Vue3中，按键修饰符可以用v-on指令来注册一个键盘事件处理函数，如下所示：

<!--  v-on  指令缩写为  @  -->
<template>
    <div>
        <input  @keydown.enter="onEnter"  />
    </div>
</template>

<script>
export  default  {
    methods:  {
        onEnter(event)  {
            //  处理回车键事件
        },
    },
};
</script>

在上面的代码中，@keydown.enter表示按下回车键时触发回调函数onEnter。如果要同时监听多个按键事件，可以使用逗号分隔多个修饰符，如@keydown.ctrl.alt.enter表示同时按下控制键、Alt键和回车键时触发回调函数。

除了enter以外，还有一些其他常用的修饰符，如：

  • .stop 阻止事件冒泡
  • .prevent 阻止默认事件
  • .capture 捕获事件
  • .self 只有在事件发生的元素是绑定事件的元素时触发
  • .once 只触发一次事件

JS事件的传播指的是事件触发后，事件如何在DOM树中传播到其它节点或元素。

JS事件传播有三个阶段：

  1. 捕获阶段：事件从DOM树的根节点一直向下传播到事件触发的节点。事件只在捕获阶段被处理，不会触发任何事件处理程序。
  2. 目标阶段：事件在触发节点的处理程序中被处理。这是事件处理程序处理事件的最终目的。
  3. 冒泡阶段：事件从目标节点开始向上冒泡，依次触发父节点的事件处理程序，直到DOM树的根节点为止。在冒泡阶段里，事件处理函数可以进行一些影响到事件流传递特性的操作。

事件传播可以被停止，也可以被阻止。

停止事件传播：调用事件对象的stopPropagation方法，可以阻止事件传播到当前节点之后的节点。

阻止默认行为：调用事件对象的preventDefault方法，可以阻止事件的默认行为。例如，如果点击了一个链接，调用preventDefault将不会跳转到链接的URL。
